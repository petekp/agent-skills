## Code Reviews & Technical Communication

Precise, specific, explains root cause AND why:

> "Variable used before definition - $ACTION is referenced on line 135 before it is defined on line 167"

> "Regression: Re-introduces staleness check... CLAUDE.md says Lock existence is authoritative"

> "No issues found... Notable improvements: Fixes the ACTION variable ordering bug from the previous review"

**Pattern:** Line numbers when relevant. Root cause + why, not just what. Acknowledge good work alongside issues.

**Pattern:** Bullet-pointed analysis, dry humor, specific craft observations. Anti-patterns: "by the numbers", "tropes upon tropes", empty spectacle.
